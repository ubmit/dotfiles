## Code style

- Only create abstractions when necessary
- Use clear names instead of comments
- Avoid `any` type
- Use inline exports: `export function foo() {}`
- Name files with kebab-case
- Colocate related code

## React

- Compose smaller components instead of large `jsx` blocks
- Avoid `useEffect`. Only use it to synchronize with external systems (browser APIs, non-React widgets, network)
- Use the frontend-design skill

## Styling

- Use Tailwind CSS v4

## Browser/Automation

- Use the agent-browser skill for browser testing or automation tasks

## Planning

- After plans, list any unresolved questions

## Git

- Use Conventional Commits for commit messages
- Prefix branches with "gui/"
- Prefer small, intentional commits

## GitHub

- Use `gh` to interact with GitHub
